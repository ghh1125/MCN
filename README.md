# MCN Workflow Studio

一个面向内容创作的网页工作台：输入创作需求，检索多平台参考内容，生成候选主题，确认后打磨剧本，再调用视频模型生成并保存视频。

当前主入口是 Streamlit 页面，重点是“人和 AI 协作式创作”，不是纯后端自动流水线。

## 主要能力

- 右侧常驻聊天区：像对话助手一样输入需求和修改意见
- 多平台搜索：支持勾选 `douyin`、`bilibili`、`xiaohongshu`
- 主题审核：不满意时可继续反馈，系统会重检索并重新出题
- 剧本打磨：支持 AI 重写，也支持直接手动编辑剧本
- 视频生成：支持接 DashScope 这类视频模型，并保存到本地
- 调试查看：保留 LLM 交互日志和原始响应，方便联调

## 页面怎么用

### 1. 左侧配置

左侧边栏有三组配置：

- `策划大模型`
  - API Key
  - URL
  - Model
- `搜索`
  - API Key
  - Provider
  - URL
- `视频生成`
  - API Key
  - 提交 URL
  - Model

这些值都只在当前网页会话中生效，不会写回 `.env`。

### 2. 中间工作区

中间区域负责展示和操作：

- 选择 `creator_id`
- 选择目标内容平台
- 勾选搜索平台
- 查看检索结果和候选主题
- 查看并编辑剧本
- 查看视频生成结果

### 3. 右侧聊天区

右侧是持续对话区：

- 第一轮直接输入创作需求
- 主题不满意时，直接输入主题修改意见
- 剧本不满意时，直接输入剧本修改意见
- 右侧也会显示当前步骤对应的确认按钮

聊天区默认显示自然语言摘要；如果某条模型回复较长，可以点开“展开查看完整内容”查看全文。

## 运行流程

1. 在右侧输入创作需求
2. 系统解析意图并联网搜索参考内容
3. 生成候选主题
4. 你确认主题，或继续给反馈改题
5. 系统生成剧本
6. 你确认剧本，或继续给反馈重写
7. 生成视频并保存到本地

如果视频生成因为时长限制报错，页面会引导你改秒数并重写剧本。

## 需要准备哪些 API

最少需要三类能力：

- 一个策划大模型 API
  - 用于意图解析、主题生成、剧本生成
- 一个搜索 API
  - 当前默认是 `tikhub`
- 一个视频生成 API
  - 当前已适配 DashScope 风格的异步视频生成

## `.env` 的角色

`.env` 现在主要保留一些非敏感默认项和开关，不再要求把 API Key 写进去。

API Key、URL、模型这些更适合频繁切换的值，都从前端输入。

## 本地视频保存

生成的视频会保存到：

- `artifacts/videos/`

这个目录已经被 `.gitignore` 忽略，不会提交到仓库。

## 说明

- 当前右侧聊天区以“用户友好”优先，显示的是自然语言摘要，不是原始 JSON
- 中间工作区仍然保留结构化结果，方便精细查看
- `LLM 实时交互日志` 主要用于联调和排错，默认可折叠
- 目前主流程以“生成视频并下载保存”为终点
